# Important Documentation Rules

Follow strict documentation rules to avoid file proliferation.

## Instructions

When completing any task, follow these documentation rules:
1. DO NOT create multiple new markdown files
2. DO NOT create additional documentation files unless absolutely necessary
3. ALWAYS update existing documentation files instead:
   - Update .claude/CLAUDE.md for project changes and setup instructions
   - Update .claude/PRD.md for product requirements and specifications
   - Update .claude/ROADMAP_MASTER_V3.md for progress tracking and completed tasks
   - update .claude/README.md to better understand the product and update it for new implementation
4. If new information is needed, add it directly to the appropriate existing document
5. Keep documentation consolidated and avoid duplication

This maintains clean project structure and prevents documentation sprawl.
